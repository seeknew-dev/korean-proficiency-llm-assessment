## 1. 과제 개요

### 과제명

**한국어 능력 평가를 위한 LLM 기반 다차원 난이도 측정 및 맞춤형 문제 생성 시스템 개발**

### 개발 목표

- 객관식 문제의 다차원 난이도 자동 측정 시스템 개발 (어휘, 문법, 독해, 논리, 기능 등)
- 수험자 응답 패턴 기반 문항 변별력 최적화 알고리즘 개발
- 언어 사용 범주별(직장, 생활, 여행 등) 맞춤형 평가 문제 세트 자동 생성 시스템 개발
- 일정 수준 이상의 한국어 능력 보유자 선발을 위한 고변별력 문제 생성 기술 개발

### 개발 방법

- LLM(Gemini 2.5 Flash) 기반 문제 생성 파이프라인 고도화
- 실제 수험자 응답 데이터 수집 및 분석을 통한 난이도 예측 모델 학습
- IRT(Item Response Theory) 기반 문항 변별도, 난이도 자동 평가 시스템 구축
- 범주별 언어 능력 측정 요소 정의 및 문제 자동 생성 알고리즘 개발
- 동형 문제 대량 생성을 통한 부정행위 방지 시스템 구축

### 연구팀 구성

- 총 참여 연구원: 3명 (여성 2명, 남성 1명, 여성 비율 66.7%)
- 연구책임자: [직접 작성] (여성/남성)
- AI/NLP 연구원: [직접 작성] (여성/남성)
- 백엔드 개발자: [직접 작성] (여성/남성)
- 한국어교육 전문가 (외부 자문): [직접 작성]

### 기대효과

- EPS-TOPIK, TOPIK, 세종학당 등 한국어 능력 시험 기관에 기술 제공
- 외국인 근로자 채용 기업의 한국어 능력 검증 비용 70% 절감
- 기업별 맞춤형 한국어 능력 평가 서비스 시장 진출
- 연 매출 10억원 달성 및 일자리 4개 창출 (3년 내)
- **여성 고용 확대**: 여성 비율 60% 이상 유지, AI 분야 여성 전문 인력 양성

## 1. 기술개발 목표

### 1.1 연구개발 필요성 및 배경

현재 외국인 근로자 채용을 위한 한국어 능력 평가는 EPS-TOPIK(고용허가제 한국어능력시험)에 의존하고 있으나, 다음과 같은 근본적인 한계가 있습니다:

1. **직군별 언어 능력 차이 미반영**:
   - EPS-TOPIK은 모든 직군(건축 시공, 기계 조립, 음식 조리, 숙박 서비스 등)에 동일한 시험을 적용
   - 건축 시공 현장의 안전 용어와 음식 조리의 위생 용어는 완전히 다른 언어 능력을 요구하나, 현행 시험은 이를 구분하지 못함
   - 결과적으로 **특정 직군에 필요한 실제 언어 능력을 정확히 측정할 수 없음**

2. **학습자의 비효율적 학습 부담**:
   - 모든 지원자가 전체 직업군에 대한 광범위한 언어를 학습해야 함
   - 자신의 희망 직군과 무관한 어휘와 표현까지 공부해야 하는 시간적, 경제적 부담
   - 실제 업무에서는 사용하지 않을 내용을 암기하는 비효율성

3. **기업의 채용 미스매치**:
   - 높은 EPS-TOPIK 점수를 받았더라도 특정 직군에서 필요한 언어 능력이 부족한 경우 발생
   - 기업은 채용 후 별도의 언어 교육을 진행해야 하는 추가 비용 발생
   - **고용허가제의 본래 취지인 "업무에 적합한 인력 선발"이 제대로 이루어지지 않음**

4. **낮은 변별력**: 단순 암기 학습으로 높은 점수를 받을 수 있어 실제 언어 능력 측정 한계

5. **제작 비용**: 고품질 문제 제작에 많은 시간과 인력 소요

본 과제는 LLM 기술을 활용하여 이러한 문제를 해결하고, **직군별로 실제 필요한 언어 능력을 정확히 측정**하며, **맞춤형 평가**가 가능한 차세대 한국어 능력 평가 시스템을 개발합니다. 이를 통해 외국인 근로자는 자신의 희망 직군에 필요한 언어만 집중적으로 학습할 수 있고, 기업은 업무에 적합한 언어 능력을 갖춘 인력을 선발할 수 있습니다.

### 1.2 최종 개발 목표

#### 핵심 목표 1: 다차원 난이도 측정 시스템

- 문제의 난이도를 1차원 점수가 아닌 **다차원 벡터**로 측정
  - 어휘 난이도, 문법 복잡도, 독해 수준, 논리적 사고력, 실용적 기능 등
- 실제 수험자 응답 데이터 기반 **난이도 예측 정확도 90% 이상** 달성
- IRT(Item Response Theory) 기반 문항 변별도, 난이도 자동 계산

#### 핵심 목표 2: 범주별 맞춤형 평가 시스템

- 언어 사용 범주(직장-건축 시공/기계 조립/음식 조리/숙박 서비스, 생활, 여행 등) 정의 및 DB 구축
- 범주별 필수 측정 요소 자동 추출 (예: 건축 시공 → 안전 용어, 명령형 표현 이해)
- 지정된 범주와 난이도에 맞는 **문제 세트 자동 생성**

#### 핵심 목표 3: 고변별력 문제 생성 엔진

- 추측에 의한 정답 가능성 **25% 이하**로 최소화 (4지선다 기준)
- 실제 학습자 오류 패턴 기반 **고급 오답 선택지(distractor)** 자동 생성
- 문항 변별도 **0.4 이상** 달성

#### 핵심 목표 4: 동형 문제 대량 생성 시스템

- 동일 난이도/변별력을 가진 서로 다른 문제를 **무한 생성**
- 동형 문제 간 난이도 편차 **±5% 이내** 보장
- 부정행위 방지를 위한 문제 풀 관리 시스템

### 1.3 정량적 성과 목표

| 항목                    | 현재 수준    | 목표 수준    | 측정 방법                      |
| ----------------------- | ------------ | ------------ | ------------------------------ |
| 난이도 예측 정확도      | 60% 이하     | 90% 이상     | 실제 정답률과 예측 정답률 비교 |
| 문항 변별도             | 0.2 이하     | 0.4 이상     | IRT 분석                       |
| 추측 정답률             | 35%          | 25% 이하     | 무작위 응답 시뮬레이션         |
| 동형 문제 난이도 일치도 | -            | ±5% 이내     | 문제 세트 간 평균 점수 비교    |
| 문제 생성 비용          | 문항당 5만원 | 문항당 1천원 | 인력 투입 시간 환산            |

## 2. 연구개발 방법

### 2.1 연구개발 추진 전략

본 과제는 **4단계 접근법**으로 진행됩니다:

**1단계**: 데이터 수집 및 분석 (1-5개월)
**2단계**: 난이도 측정 모델 개발 (5-10개월)
**3단계**: 범주별 평가 시스템 개발 (10-14개월)
**4단계**: 통합 및 검증 (14-18개월)

### 2.2 세부 연구개발 방법

#### 2.2.1 1단계: 데이터 수집 및 분석 (1-5개월)

**목표**: 문제 난이도 측정을 위한 실제 수험자 응답 데이터 확보

**방법**:

1. **기존 EPS-TOPIK 문제 활용**
   - 자체 개발한 문제 생성 파이프라인으로 생성된 1,000개 이상의 문제 활용
   - 2026년 2월 오픈 예정인 서비스를 통해 실제 학습자 응답 데이터 수집

2. **데이터 수집 체계 구축**
   - PostgreSQL 기반 응답 데이터 저장소 구축
   - 수험자 ID, 문제 ID, 응답 시간, 정답 여부, 선택 선택지 등 수집
   - 개인정보 비식별화 처리

3. **응답 패턴 분석**
   - 문제별 정답률, 응답 시간 분포 분석
   - 수험자 수준별(초급/중급/고급) 정답률 차이 분석
   - 오답 선택지별 선택 비율 분석 (distractor 효과성 측정)

**검증 방법**:

- 최소 500명 이상의 학습자로부터 50,000개 이상의 응답 데이터 수집
- 데이터 품질 검증: 무작위 응답 패턴 제거, 비정상적 응답 시간 필터링

#### 2.2.2 2단계: 난이도 측정 모델 개발 (5-10개월)

**목표**: 문제의 다차원 난이도를 자동으로 측정하는 AI 모델 개발

**방법**:

1. **다차원 난이도 정의**
   - 어휘 난이도: 사용된 단어의 빈도, 급수 (1급~6급)
   - 문법 복잡도: 문장 길이, 복문 사용, 고급 문법 요소
   - 독해 수준: 지문의 추상성, 함축적 의미 포함 여부
   - 논리적 사고: 추론, 비교, 인과관계 파악 필요 정도
   - 실용적 기능: 실생활 맥락 이해도 (격식체/비격식체, 상황 판단 등)

2. **IRT(Item Response Theory) 기반 분석**
   - 1단계에서 수집한 응답 데이터를 IRT 모델에 적용
   - 문항 난이도(difficulty), 변별도(discrimination), 추측도(guessing) 계산
   - Python `mirt` 또는 `pyirt` 라이브러리 활용

3. **LLM 기반 난이도 예측 모델**
   - Gemini 2.5 Flash를 활용하여 문제 텍스트로부터 난이도 요소 추출
   - Few-shot learning: IRT로 측정된 실제 난이도를 학습 데이터로 활용
   - Fine-tuning: 한국어 문제 특화 난이도 예측 모델 학습
   - AI Pipeline을 활용한 자동 학습 파이프라인 구축

4. **오답 선택지 품질 평가**
   - Distractor 매력도 측정: 각 오답 선택지가 실제로 선택된 비율 분석
   - 너무 쉽게 배제되는 선택지 자동 탐지 및 개선 제안
   - 오답 선택지의 교육학적 타당성 평가 (흔한 오류 패턴 반영 여부)

**검증 방법**:

- 예측 난이도와 실제 정답률 간 상관계수 0.85 이상
- 교차 검증: 신규 문제의 예측 정답률과 실제 정답률 비교
- 전문가 평가: 한국어교육 전문가 3인의 주관적 난이도 평가와 비교

#### 2.2.3 3단계: 범주별 평가 시스템 개발 (10-14개월)

**목표**: 언어 사용 환경별 맞춤형 평가 문제 자동 생성

**방법**:

1. **범주 정의 및 DB 구축**
   - 직장: 건축 시공, 토목 시공, 기계 조립, 금속 가공, 음식 조리, 숙박 서비스, 작업장 관리 등
   - 생활: 쇼핑, 병원, 은행, 관공서, 이웃 관계 등
   - 여행: 공항, 호텔, 관광지, 식당, 교통수단 등
   - 범주별 필수 어휘 리스트 구축 (예: 건축 시공 → 안전모, 비계, 추락, 안전수칙 등)
   - 범주별 필수 문법/표현 (예: 음식 조리 → 주문 받기, 손님 응대 표현 등)

2. **범주별 측정 요소 정의**
   - 각 범주에서 평가해야 할 언어 능력 요소 정의
   - 예: 건축 시공 → 안전 용어 이해, 명령형 표현 이해, 위험 상황 판단
   - 예: 기계 조립 → 공정 용어 이해, 작업 지시 이해, 불량 보고 표현
   - 예: 숙박 서비스 → 손님 응대 표현, 예약 관련 용어, 시설 안내 능력

3. **맞춤형 문제 생성 알고리즘**
   - Gemini 2.5 Flash에 범주별 지식을 프롬프트로 주입
   - 범주별 필수 어휘, 문법, 상황을 포함한 문제 자동 생성
   - 지정된 난이도 벡터에 맞춰 문제 난이도 조정
   - 예: "건축 시공, 중급, 안전 용어 중심, 명령형 표현 이해" → 해당 조건에 맞는 문제 생성

4. **문제 세트 구성 알고리즘**
   - 목표 점수 분포를 입력하면 이에 맞는 문제 조합 자동 선정
   - 범주 내 다양한 하위 주제를 균형 있게 포함
   - 문법/어휘/독해/실용 영역의 비율 조정 가능

**검증 방법**:

- 범주별 전문가 검증: 건축 시공 종사자, 기계 조립 종사자, 음식 조리 종사자 등의 실제 피드백 수집
- 문제의 범주 적합성 평가: 전문가 3인의 블라인드 평가 (정확도 90% 이상)
- A/B 테스트: 범주별 맞춤 문제와 일반 문제의 예측력 비교

#### 2.2.4 4단계: 통합 및 검증 (14-18개월)

**목표**: 전체 시스템 통합 및 실전 환경 검증

**방법**:

1. **동형 문제 대량 생성 시스템**
   - 동일 난이도/변별력을 가진 문제를 무한 생성하는 알고리즘 개발
   - Paraphrasing 기반: 표면적 표현은 다르지만 측정 능력은 동일한 문제 생성
   - 어휘/구문 변형을 통한 변형 생성 (예: "안전모를 착용하세요" → "작업 시 머리 보호구를 반드시 쓰세요")
   - 문제 풀 관리 시스템: 난이도/영역/유형별 문제 DB 구축, 노출 이력 관리

2. **통합 평가 플랫폼 구축**
   - React Native (Expo) 기반 크로스 플랫폼 앱에 통합
   - 컨테이너 기반 API 서버을 통한 백엔드 API 배포
   - AI Pipeline을 통한 문제 생성 자동화

3. **실전 검증**
   - 베타 테스터 100명 이상 모집하여 실제 평가 진행
