# 3. 선행연구개발

**해당 사항 없음**

본 연구개발기관은 중기부 R&D 사업 또는 타 부처 지원 R&D 사업을 수행한 이력이 없습니다.

다만, **자체 연구개발 실적**은 다음과 같습니다:

## 자체 개발 실적

**과제명**: EPS-TOPIK 객관식 문제 자동 생성 시스템 개발
**기간**: 2025년 6월 ~ 2026년 1월 (8개월)
**투자 규모**: 자체 자금 5,000만원
**개발 내용**:

- LLM 기반 문제 생성 파이프라인 구축
- EPS-TOPIK 공식 교재를 기반으로 유형별 문제 생성 가능
- PostgreSQL 기반 문제 데이터베이스 구축
- React Native 기반 크로스 플랫폼 모바일 앱 개발

**현재 한계점**:

- 생성된 문제의 난이도 측정이 부정확
- 실제 수험자 응답 데이터 부족으로 문항 변별력 검증 불가
- 범용 문제만 생성 가능 (직군별 맞춤형 평가 불가)

**본 과제와의 차별성**:
본 과제는 자체 개발한 문제 생성 파이프라인을 기반으로, **난이도 측정의 정확성**, **직군별 맞춤형 평가**, **학습자 맞춤형 추천 시스템**이라는 핵심 기술을 새롭게 개발하는 것입니다. 기존 시스템은 "문제 생성"에 초점을 맞췄다면, 본 과제는 "정확한 능력 측정", "직군별 평가", "맞춤형 학습 지원"이라는 고부가가치 기술 개발을 목표로 합니다.

## 본 과제 수행 시 선행연구개발 결과 활용계획

선행연구개발에서 구축한 핵심 인프라와 기술을 본 과제의 기반으로 활용하여, 개발 기간 단축 및 기술 고도화를 동시에 달성할 계획입니다.

### 1. LLM 기반 문제 생성 파이프라인 활용

**선행연구개발 결과**:

- LLM을 활용한 EPS-TOPIK 읽기, 듣기 문제 생성 파이프라인
- EPS-TOPIK 공식 교재 기반 문제 생성 프롬프트 템플릿

**본 과제 활용 계획**:

- **1단계 (데이터 수집)**: 기존 생성 파이프라인으로 초기 문제 세트 구축하여 응답 데이터 수집 시작
- **2단계 (난이도 측정)**: 기존 생성된 문제에 대한 실제 응답 데이터를 활용하여 IRT 모델 학습 및 검증
- **3단계 (직군별 평가)**: 기존 프롬프트 템플릿을 직군별로 확장하여 맞춤형 문제 생성
- **4단계 (맞춤형 추천)**: 기존 생성 엔진을 개선하여 동형 문제 대량 생성 시스템으로 발전

### 2. React Native 모바일 앱 활용

**선행연구개발 결과**:

- Expo 기반 크로스 플랫폼 모바일 앱 (iOS, Android)
- 기본적인 문제 풀이 및 정답 확인 기능
- **2026년 2월 정식 오픈 예정**

**본 과제 활용 계획**:

본 과제는 **2026년 2월 오픈 예정인 자사 한국어 교육 서비스를 연구개발 실험실로 활용**하여, 실제 사용 환경에서 연구를 수행합니다.

- **1단계 (데이터 수집)**: 앱을 통한 실제 사용자 응답 데이터 수집 플랫폼으로 활용
  - **자사 서비스를 연구개발 실험실로 활용**: 2026년 2월 오픈 후 실제 학습자로부터 응답 데이터 수집
  - 서비스 정식 오픈 후 50,000건 이상 응답 데이터 확보
  - 개인정보 비식별화 처리 및 분석용 데이터 추출
- **4단계 (맞춤형 추천 시스템)**: 자사 서비스 기반 장기 학습 효과 검증
  - **실험 환경**: 실제 한국어 학습자를 대상으로 맞춤형 추천 시스템 효과 검증
  - 실험군 vs 대조군 3개월 학습 효과 비교 (각 50명 이상)
  - 실제 사용 환경에서의 지속적인 알고리즘 개선 및 검증
- **5단계 (통합 검증)**: 통합 평가 및 학습 플랫폼의 프론트엔드로 활용
  - 평가 모드 + 학습 모드 UI/UX 추가 개발
  - 맞춤형 추천 알고리즘 적용 인터페이스 구현
  - 연구 종료 후 즉시 상용화 가능한 검증된 서비스 확보

### 3. PostgreSQL 데이터베이스 활용

**선행연구개발 결과**:

- 문제 데이터베이스 스키마 (questions, options, answers)
- 기본적인 문제 관리 기능

**본 과제 활용 계획**:

- **1단계 (데이터 수집)**: 응답 데이터 수집을 위한 스키마 확장
  - user_responses, test_sessions 테이블 추가
  - 응답 시간, 오답 선택지, 수험자 메타데이터 저장
- **2단계 (난이도 측정)**: IRT 분석 결과 저장
  - item_parameters 테이블 추가 (난이도, 변별도, 추측도)
  - 다차원 난이도 벡터 저장 (vocabulary, grammar, reading, logic, function)
- **3단계 (직군별 평가)**: 직군별 문제 분류 체계 구축
  - job_categories, job_specific_questions 테이블 추가
  - 직군별 필수 어휘 데이터베이스 구축
- **4단계 (맞춤형 추천)**: 학습자 프로파일 및 추천 이력 저장
  - learner_profiles, recommendation_history 테이블 추가

### 4. GCP 인프라 활용

**선행연구개발 결과**:

- 컨테이너 기반 API 서버 기반 API 서버 운영 경험
- AI Pipeline 기본 구축 경험

**본 과제 활용 계획**:

- **전 단계 공통**: 안정적인 서비스 운영 및 확장
  - 컨테이너 기반 API 서버: API 서버 배포 및 자동 스케일링
  - 관리형 데이터베이스(PostgreSQL) 운영
  - Cloud Storage: 문제 데이터 및 모델 저장
- **2단계 (난이도 측정)**: AI Pipeline을 활용한 자동 학습 파이프라인
  - 응답 데이터 수집 → IRT 분석 → 난이도 예측 모델 재학습 자동화
  - LLM 기반 난이도 예측 모델 Few-shot learning 자동화
- **5단계 (통합 검증)**: 프로덕션 배포 및 성능 최적화
  - 컨테이너 기반 API 서버 성능 모니터링 및 비용 최적화
  - 콘텐츠 전송 네트워크(CDN)을 통한 정적 리소스 캐싱

### 활용 효과

**개발 기간 단축**:

- 문제 생성 파이프라인, 앱, 인프라가 이미 구축되어 있어 **초기 개발 기간 약 3~4개월 단축**
- 1단계 데이터 수집을 즉시 시작 가능 (앱 재개발 불필요)
- 2026년 2월 서비스 오픈 시점부터 연구 착수 가능

**기술 고도화**:

- 기존 시스템의 한계점(난이도 측정, 직군별 평가, 학습 지원)을 명확히 파악한 상태에서 개발 시작
- 실제 운영 경험을 바탕으로 한 현실적인 요구사항 반영
- **실제 사용 환경에서의 지속적 검증**: 자사 서비스를 실험실로 활용하여 연구 신뢰성 확보

**비용 효율성**:

- 신규 인프라 구축 비용 절감 (기존 GCP 인프라 활용)
- 앱 개발 비용 절감 (기존 앱 개선만으로 충분)
- 별도 테스트 환경 구축 불필요 (자사 서비스 활용)
- **예상 절감액: 약 2,000만원** (인프라 구축 1,000만원 + 앱 개발 1,000만원)

**연구개발 실험실로서의 가치**:

- 실제 한국어 학습자를 대상으로 한 대규모 실험 가능
- 연구 종료 후 즉시 상용화 가능한 검증된 기술 확보
- 연구 기간 중 지속적인 데이터 축적 및 알고리즘 개선
